# 钉钉消息发送格式规范

## 核心规则

1. **双换行分段** — 每行之间必须有一个空行，否则钉钉会合并为一行
2. **禁用表格** — 钉钉单聊不渲染 Markdown 表格，会显示原始 `|---|` 符号
3. **emoji 标记** — 行首放 emoji 辅助区分内容类型
4. **符号对齐** — 用 `▸` 分隔列，`——` 做分隔线

## 发送方式

### 文本消息

```bash
# 短消息直接写
dws chat message send --user <userId> --title "标题" --text "内容" -f json -y

# 长消息用文件
dws chat message send --user <userId> --title "标题" --text "@C:/path/to/msg.txt" -f json -y
```

### 换行规则

| 方式 | 效果 |
|------|------|
| 文件中的双换行（空行分隔） | ✅ 每段独立显示 |
| 文件中的单换行 | ❌ 被当作空格合并 |
| `\n` 字面量 | ❌ 不解析 |

## 图片消息

### 限制
- 单聊发图片暂不完整支持，用钉盘链接替代
- 群聊可用 `--group` + `--msg-type image` + `--media-id`

### 替代方案：上传钉盘发链接

```bash
# 上传
dws drive upload --file ./summary.png -f json -y

# 发送链接
dws chat message send --user <userId> --title "查看报告" --text "报告链接: <url>" -f json -y
```

## 场景选择

| 场景 | 推荐方式 |
|------|---------|
| 简短通知（<5行） | 直接 `--text "内容"` |
| 结构化长文本 | 文件传入 `--text "@file.txt"`，双换行分段 |
| 需要表格/精确排版 | 生成图片 → 上传钉盘 → 发链接 |
